# DP 初学者友好复用方案 v0.1

## 1. 当前框架是否足够

结论：当前“统一接口层 + 标准容器 + 模块格式 + 拼接食谱”的方向已经能较好支撑模块化拼接，但 DP 需要再加一层。

原因：

- 图论、数据结构天然可以通过 `Graph`、`树状数组`、`SegmentTree` 等接口拼接。
- DP 的难点不是接口，而是“状态怎么设”和“这题像哪个模型”。
- 对基础薄弱且时间紧的考生，DP 卷不能只写模板，必须做成“模型识别与复用系统”。

因此 DP 部分应采用三层结构：

```text
DP 路由层：看到题面特征，先匹配已有模型
DP 模型层：每个模型给固定状态句式、转移模板、初始化、答案位置
DP 造模层：匹配不上时，从暴力 DFS -> 记忆化 -> 表推 DP
```

目标不是理解所有 DP 原理，而是做到：

```text
题面 -> 模型候选 -> 套状态句式 -> 抄转移骨架 -> 先交记忆化/部分分 -> 有时间改表推
```

## 2. DP 使用总流程

考场遇到疑似 DP，按下面顺序走。

### Step 1：先看数据范围

```text
n <= 20：优先 DFS / 状压 / 记忆化
n <= 40：考虑折半枚举，也可能是 DP
n <= 300：区间 DP / Floyd 类 O(n^3)
n <= 3000/5000：O(n^2) 线性 DP / LCS / LIS 朴素
n <= 2e5：O(n log n) / O(n)，DP 通常要优化或换数据结构
```

### Step 2：识别处理对象

```text
序列：dp[i] / dp[i][state]
容量/预算：背包 dp[j] / dp[i][j]
两个序列：dp[i][j]
区间：dp[l][r]
树：dp[u][state]
DAG：拓扑序后 dp[u]
集合：dp[mask] / dp[mask][last]
数字上界：dfs(pos, tight, leading, state)
网格：dp[i][j] / BFS
```

### Step 3：先套模型，不要空想

优先翻“DP 模型卡片”。如果能匹配，就直接照卡片写。

### Step 4：匹配不上，写暴力 DFS

写成：

```text
状态 = 当前处理到哪里 + 当前还剩/已用资源 + 必须记住的历史
选择 = 当前一步能做什么
转移 = 做选择后状态怎么变
```

### Step 5：先记忆化，后表推

如果状态顺序想不清，保留记忆化。记忆化比错误表推更适合弱基础考场策略。

## 3. DP 模型卡片统一格式

每个 DP 模型必须固定格式，不自由发挥。

```text
模型编号：
模型名称：
题面触发词：
数据范围信号：
状态句式：
初始化：
转移模板：
答案位置：
循环顺序：
常见变体：
暴力 DFS 版本：
记忆化版本：
表推版本：
常见坑：
可拼接模块：
```

其中最重要的是“状态句式”。例如：

```text
dp[i][j] 表示：处理完前 i 个物品，容量为 j 时的最大价值。
```

要求状态句式必须能回答：

```text
处理到哪里？
还记住什么？
dp 值代表最大、最小、方案数，还是可不可行？
```

## 4. DP 模型库优先级

### Tier 0：必须能快速复用

这些应该放在 DP 卷最前面。

| 模型 | 触发词 | 状态句式 |
|---|---|---|
| 线性 DP | 前 i 个、最大收益、最少代价、不相邻 | `dp[i]` 表示处理到第 i 个的最优值 |
| 选/不选 DP | 每个元素选或不选 | `dp[i]` 或 `dp[i][j]` 表示前 i 个选择后的结果 |
| 0/1 背包 | 每个物品最多一次，容量限制 | `dp[j]` 表示容量 j 的最大价值 |
| 完全背包 | 每种物品无限次 | `dp[j]` 表示容量 j 的最大价值 |
| 分组背包 | 每组最多选一个 | `dp[j]` 表示处理若干组后容量 j 的最优值 |
| LCS | 两个序列，最长公共子序列 | `dp[i][j]` 表示前 i 和前 j 的 LCS |
| 编辑距离 | 插入、删除、替换 | `dp[i][j]` 表示前 i 变成前 j 的最小操作 |
| LIS | 最长递增子序列 | `d[len]` 表示长度 len 的最小结尾 |
| 区间 DP | 合并、删除、括号、回文、两端取 | `dp[l][r]` 表示区间 `[l,r]` 的最优值 |
| 状压 DP | n<=20，集合，访问所有点 | `dp[mask][last]` 表示访问集合 mask 且最后在 last |
| 记忆化 DFS | 状态复杂、顺序不清 | `dfs(state)` 返回该状态后续最优值 |

### Tier 1：应该准备

| 模型 | 触发词 | 状态句式 |
|---|---|---|
| 树形 DP | 树上选择、覆盖、最大独立集 | `dp[u][state]` 表示 u 子树在状态 state 下的最优值 |
| DAG DP | 依赖关系、有向无环、路径计数 | `dp[u]` 表示到/从 u 的最优值或方案数 |
| 网格 DP | 只能向右/下、路径数、最小路径和 | `dp[i][j]` 表示到格子 `(i,j)` 的结果 |
| 计数 DP | 方案数、取模 | `dp[state]` 表示方案数量 |
| 可行性 DP | 能否达到、是否存在 | `dp[state]` 为 bool |
| 数位 DP | 1..N 中满足条件数量 | `dfs(pos, tight, leading, state)` |

### Tier 2：打印备用

| 模型 | 触发词 |
|---|---|
| 概率/期望 DP | 期望次数、概率 |
| 博弈 DP | 必胜/必败 |
| 单调队列优化 DP | `dp[i]=min(dp[j]+cost)` 且 j 在滑窗 |
| 斜率优化 DP | 高级，低优先级 |

## 5. DP 路由表

纸质资料中应有一张“一眼路由表”。

```text
看到“每个东西选/不选” -> 0/1 背包或子集 DFS
看到“容量/重量/预算” -> 背包
看到“无限个/可以重复选” -> 完全背包
看到“每组只能选一个” -> 分组背包
看到“两个字符串/两个序列” -> LCS / 编辑距离
看到“最长递增/不下降子序列” -> LIS
看到“区间合并/从两端取/删除回文” -> 区间 DP
看到“n <= 20 且集合相关” -> 状压 DP
看到“树上选点/覆盖/染色” -> 树形 DP
看到“依赖关系/有向无环图” -> 拓扑 DP
看到“上界很大，按数字位限制” -> 数位 DP
看到“最少步数且状态变化” -> BFS 状态搜索，不一定是 DP
看到“重复状态明显但顺序想不清” -> 记忆化 DFS
```

## 6. 状态设计的初学者句式

给用户一组可直接套的句式。

### 6.1 序列句式

```text
dp[i] 表示：处理完前 i 个元素时，____ 的最优值/方案数/是否可行。
```

常见填空：

```text
最大收益
最小代价
方案数
最后一步选择状态
```

### 6.2 背包句式

```text
dp[j] 表示：当前已处理若干物品，容量/花费为 j 时的最大价值/方案数/是否可行。
```

循环方向：

```text
0/1 背包：j 从大到小
完全背包：j 从小到大
分组背包：每组用 ndp
```

### 6.3 区间句式

```text
dp[l][r] 表示：只考虑区间 [l, r] 时，____ 的最优值/方案数。
```

遍历顺序：

```text
按 len 从小到大
枚举 l
计算 r = l + len - 1
枚举分割点 k
```

### 6.4 状压句式

```text
dp[mask][last] 表示：已经选择/访问集合 mask，最后一个元素是 last 时的最优值。
```

常见选择：

```text
枚举下一个未选元素 v
nmask = mask | (1 << v)
```

### 6.5 树形句式

```text
dp[u][state] 表示：只考虑 u 的子树，并且 u 自己处于 state 状态时的最优值/方案数。
```

常见状态：

```text
选/不选 u
u 被父亲覆盖 / 被儿子覆盖 / 未覆盖
u 的颜色
```

### 6.6 DAG 句式

```text
dp[u] 表示：到达 u 的最优值/方案数，或从 u 出发能得到的最优值。
```

固定流程：

```text
建有向图
拓扑排序
按拓扑序转移
```

## 7. 匹配不上模型时的造模流程

### 7.1 三问版

考场最短版：

```text
1. 我现在在哪里？        -> i / pos / u / l,r / mask
2. 我还必须记住什么？    -> j / last / cnt / tight / used
3. 我下一步能选什么？    -> choice
```

### 7.2 五问完整版

```text
1. 状态表示什么？
2. 初始状态是什么？
3. 终止条件是什么？
4. 每一步有哪些选择？
5. 答案从哪里取？
```

### 7.3 先写 DFS

```cpp
ll dfs(State s) {
    if (is_end(s)) return base_value(s);

    ll ans = -LINF; // 最大值题
    // ll ans = LINF; // 最小值题
    // ll ans = 0;    // 计数题

    for (Choice c : get_choices(s)) {
        if (!can_choose(s, c)) continue;
        State t = go(s, c);
        ans = max(ans, gain(s, c) + dfs(t));
    }

    return ans;
}
```

### 7.4 再加记忆化

稳妥版使用 `map<tuple<...>, ll>`：

```cpp
map<tuple<int,int,int>, ll> memo;

ll dfs(int i, int j, int last) {
    auto key = make_tuple(i, j, last);
    if (memo.count(key)) return memo[key];

    if (/* end */) return memo[key] = 0;

    ll ans = -LINF;
    // enumerate choices

    return memo[key] = ans;
}
```

如果状态范围明确，再改数组：

```cpp
vector<vector<ll>> memo(n + 1, vector<ll>(M + 1, -LINF));
vector<vector<int>> vis(n + 1, vector<int>(M + 1, 0));
```

### 7.5 最后再表推

只有当满足以下条件时才建议表推：

```text
状态范围清晰
循环顺序清晰
每个状态依赖前面已算好的状态
内存能开
```

否则记忆化就是可接受版本。

## 8. DP 得分梯度

DP 题建议每题按四档提交。

### Version 0：特判和合法输出

```text
n=1
空输入/空集合
全相同
无解输出
```

### Version 1：暴力 DFS

适用：

```text
n <= 20
状态少
枚举选择能写出来
```

### Version 2：记忆化 DFS

适用：

```text
重复状态明显
状态可以用参数描述
表推顺序想不清
```

### Version 3：模型 DP

适用：

```text
匹配背包/区间/树形/状压/DAG 等现成模型
数据范围要求更高
```

### Version 4：DP + 数据结构优化

低优先级，冲高分：

```text
前缀和优化转移
单调队列优化
树状数组/SegmentTree 优化 max/min 查询
滚动数组省空间
```

## 9. DP 与其他模块的拼接

DP 不是孤立的，应明确可拼接模块。

```text
DP + PrefixSum
  -> 区间和 cost(l,r) 快速计算

DP + Compressor
  -> 值域作为状态时先压缩

DP + 树状数组/SegmentTree
  -> 优化“前面某范围最大/最小”的转移

DAG DP + Graph + Topo
  -> 有向无环依赖问题

树形 DP + Graph
  -> 树上选择、覆盖、染色

状压 DP + Floyd/Dijkstra
  -> TSP/访问关键点

记忆化 DFS + map<tuple>
  -> 状态复杂但数量不大的部分分
```

## 10. DP 卷排版建议

DP 卷不应按“原理章节”排，而应按考场动作排。

```text
DP-00：DP 总流程
DP-01：DP 路由表
DP-02：状态句式库
DP-03：DFS -> 记忆化 -> 表推升级图
DP-03B：状态增维路由
DP-04：线性 DP
DP-05：选/不选 DP
DP-06：0/1 背包
DP-07：完全背包
DP-08：分组背包
DP-09：LCS
DP-10：编辑距离
DP-11：LIS
DP-12：网格 DP
DP-13：区间 DP
DP-14：树形 DP
DP-15：DAG DP
DP-16：状压 DP
DP-17：数位 DP
DP-18：DP + 数据结构优化

```

```text

3. DFS -> 记忆化 -> DP 升级图
4. 状态增维路由
```

这几页决定弱基础考生能不能真的用起来。

## 11. 对执行计划的进一步要求

后续生成 DP 卷时，每个 DP 模块必须额外给：

```text
暴力 DFS 入口
记忆化版本
表推版本
状态句式
答案位置
循环顺序
初始化原因
能拿哪些数据范围
能和哪些模块拼接
```

尤其是“初始化原因”要用一句话讲清楚，不写证明。例如：

```text
dp[0][0] = 0：还没选任何物品、容量为 0 时价值为 0。
其他为 -LINF：表示暂时不可达。
```

## 12. 一句话结论

DP 部分的初学者友好设计，不是把所有 DP 讲明白，而是给用户一条可执行路线：

```text
先查模型卡片；
能匹配就套状态句式；
不能匹配就写 DFS；
DFS 能重复就加 memo；
状态顺序清楚再表推；
不会满分也先交部分分。
```
