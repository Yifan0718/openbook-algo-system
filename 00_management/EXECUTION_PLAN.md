# 北京中关村学院夏令营机考纸质开卷资料包 - 执行方案 v0.1

## 0. 项目目标

本项目不是制作“算法教材”，而是制作一套可在 3 小时、3 道题、开卷但不能联网/不能使用生成式 AI 的机考中使用的纸质版算法作战系统。

核心目标：

1. 5 分钟内从题面和数据范围定位候选算法。
2. 10-20 分钟内先写出暴力/部分分版本。
3. 能通过路由表找到升级路线：暴力 -> 记忆化 -> DP/图论/数据结构正解。
4. C++17 模块尽量自包含、接口稳定、少魔改、可直接抄写拼接。
5. 资料宁滥勿缺，但必须高可导航，避免考场翻资料迷路。

## 1. 已核对的考试约束

依据用户提供信息和源 PDF 视觉核对，当前约束如下：

- 考试内容：编程基础、算法与数据结构。
- 编程语言：C/C++、Java、Python，建议主力使用 C++17。
- 考试时长：3 小时。
- 题量：3 道题。
- 提交：每道题可进行 32 次有效提交，取该题多次提交中的最高分。
- 资料：考试过程中可以翻阅提前准备好的纸质资料。
- 禁止：搜索引擎等开放式工具、ChatGPT/Copilot 等生成式 AI 或需要网络通信的代码补全。
- 程序：使用标准输入输出，不应使用文件操作。
- 评测环境要点：Linux；C/C++ 编译器包含 g++/gcc，页面示例中可选 g++ with std17。

策略含义：

- 必须利用“最高分”机制：每题先交部分分，再逐步升级。
- 纸质资料必须覆盖暴力、记忆化、小数据精确解和兜底合法输出。
- C++ 模板不使用 `freopen`、`#pragma GCC optimize`、系统调用或依赖网络/外部库的内容。

## 3. 资料总体结构

### 第 -1 卷：统一接口与拼接规范

用途：把资料从“模板大全”升级成“可拼接系统”。所有算法模块先面向统一容器，而不是直接面向题面原始输入。

必须包含：

- 全局类型：`ll`、`pii`、`INF/LINF/MOD`。
- 全局索引：图/数组/DP 默认 1-index，区间默认闭区间 `[l, r]`，字符串默认 0-index 并单独标注。
- 统一函数命名：`init/build/add/add_edge/setv/range_add/query/prefix/at`。
- 标准 `Graph`：同时维护邻接表 `G.g` 和逻辑边表 `G.edges`。
- 标准数组：`vector<ll> a(n + 1)`。
- 标准查询：`Query{l, r, id, x}`。
- 标准坐标压缩：压缩后编号从 1 开始，可直接接 树状数组/Segment Tree。
- 标准状态：`State = {进度, 资源, 记忆}`。
- 模块拼接食谱：Graph + Dijkstra，Graph + DSU/Kruskal，Compressor + 树状数组，暴力 DFS + memo + DP 等。

交付标准：

- 放在所有卷之前，作为“资料接口协议”。
- 后续每个模块必须声明自己依赖哪个标准容器。
- 允许代码略长，但减少考场魔改。

### 第 0A 卷：题面路由与拼接食谱

用途：解决“知道有模板，但不知道用哪个、怎么接”的问题。这一卷必须从题面特征和数据范围出发，而不是从算法名出发。

必须包含：

- 题面信号 -> 模型 -> 主模块 -> 辅助模块的总路由表。
- 数据范围 -> 复杂度 -> 可用算法的预算表。
- 操作类型 -> 数据结构的选择表。
- 模块接口契约：输入依赖、输出能力、下游可接、不能处理、初始化顺序。
- 常见拼接食谱：场景、判断信号、模块组合、拼接顺序、伪代码骨架、最小验错、部分分版本、升级版本。

交付标准：

- 前置于具体算法卷。
- 每个食谱要说明至少一个暴力/部分分版本和一个升级版本。
- 每个食谱要有 3-5 个最小验错点。

### 第 0 卷：考场作战手册

用途：考场前 10 分钟和每题开始时使用。

必须包含：

- 规则摘要。
- C++17 主骨架。
- 3 小时作战流程。
- 每题 32 次提交策略。
- 复杂度与数据范围对照表。
- 总题型路由表。
- 提交前检查清单。
- 完全不会时的兜底策略。

交付标准：

- 10-20 页内。
- 表格优先，少讲原理。
- 所有关键表放在最前面。

### 第 1 卷：C++17/STL 快查

用途：解决“算法大概知道，但 C++ 写不稳”的问题。

必须包含：

- 输入输出、多组数据、读到 EOF、整行读取。
- `vector/string/pair/tuple/sort/lambda`。
- `lower_bound/upper_bound`。
- `priority_queue` 最大堆/最小堆。
- `queue/deque/stack`。
- `set/multiset/map/unordered_map`。
- `bitset` 和位运算。
- 坐标压缩。
- `long long`、`__int128`、溢出处理。

交付标准：

- 每个 STL 条目有 3 件事：什么时候用、最短代码、常见坑。

### 第 2 卷：暴力、记忆化与部分分

用途：保证每题先活下来，并提供从暴力 DFS 到记忆化搜索的低成本升级路径。它是 DP 卷的前置桥梁：先用搜索表达“状态 + 选择 + 转移”，再用缓存显著减少重复计算。

必须包含：

- 全排列。
- 组合枚举。
- 子集枚举。
- 子集的子集枚举。
- DFS 回溯。
- 剪枝。
- 记忆化搜索作为独立核心章节。
- 暴力 DFS -> 记忆化搜索的改造手册。
- `vector`/数组 memo、`map<tuple,...>` memo、`unordered_map` 编码 memo 三类模板。
- 什么时候能缓存：状态必须完整描述后续问题。
- 什么时候不能缓存：状态遗漏历史、返回值依赖路径副作用、全局变量未纳入状态。
- BFS 状态搜索。
- 折半枚举。
- 小数据精确 + 大数据兜底结构。
- 如何写合法兜底输出。

交付标准：

- 每个模板必须标注能吃下的大致数据范围。
- 每个模板必须给出“如何升级”的方向。
- 记忆化章节必须能让考生直接把一个暴力 `dfs(...)` 改成 `dfs(...) + memo`。
- 记忆化章节必须说明与 DP 卷的关系：记忆化是抢分版本，表推 DP 是进一步升级版本，不强制每题都改表推。

### 第 3 卷：DP 题型字典

用途：把 DP 当成可路由、可套用的状态设计工具。

必须包含第一页：

- DP 通用五问：
  1. 我处理到哪里了？
  2. 当前还要记住什么？
  3. 每一步选择是什么？
  4. base case 是什么？
  5. 答案在哪里？

必须包含模块：

- 线性 DP。
- 0/1 背包、完全背包、多重背包、分组背包。
- LCS、编辑距离、LIS。
- 区间 DP。
- 树形 DP。
- DAG DP。
- 状压 DP。
- 数位 DP。
- 概率/期望 DP。
- 博弈 DP 低优先级补充。
- 记忆化 DFS 作为复杂 DP 的入口。

交付标准：

- 不追求证明，追求状态设法、转移方向、初始化、答案位置。
- 每个 DP 模块都要有“暴力递归版本”和“记忆化/表推版本”的对应关系。

### 第 4 卷：数据结构

用途：处理区间、动态查询、排序统计和维护类题目。

必须包含：

- 前缀和、一维/二维。
- 差分、一维/二维。
- 双指针、滑动窗口。
- 单调栈。
- 单调队列。
- 堆。
- 并查集。
- 树状数组。
- 线段树短版：单点修改 + 区间查询。
- 线段树完整版：区间修改 + 懒标记。
- Sparse Table。
- 坐标压缩。

交付标准：

- 数据结构分短版和完整版。
- 优先给“调用方式”，避免只贴类定义。

### 第 5 卷：图论

用途：图题模块化套用。

必须包含：

- 图建模方式：无权、有权、有向、无向。
- DFS/BFS 连通块。
- 无权最短路 BFS。
- Dijkstra。
- Floyd。
- Bellman-Ford/SPFA。
- 拓扑排序。
- DAG 最短/最长路。
- DSU/Kruskal。
- 二分图染色。
- 二分图匹配。
- SCC。
- LCA。
- Dinic 最大流低优先级补充。

交付标准：

- 每个模块必须写清楚有向/无向、是否允许负边、点编号、INF 类型。

### 第 6 卷：数学与字符串

用途：覆盖常见数论、组合和字符串模板。

数学必须包含：

- `gcd/lcm`。
- 快速幂。
- 模运算。
- 逆元。
- 组合数预处理。
- 筛法。
- 质因数分解。
- 矩阵快速幂。
- 容斥基础。

字符串必须包含：

- `string` 常用操作。
- KMP。
- Z 函数。
- Trie。
- Rolling Hash。
- LCS/编辑距离交叉引用到 DP 卷。
- Manacher/AC 自动机作为低优先级补充。

交付标准：

- 数学模块必须写“模数是否为质数”的判断分支。
- 字符串模块必须写 0-index/1-index 选择。

### 第 7 卷：综合路由与模拟训练

用途：训练如何从题面路由到模块。

必须包含：

- 关键词 -> 模块。
- 数据范围 -> 复杂度 -> 模块。
- 题型混合时的优先判断。
- 10-20 个模拟题面摘要，每个只训练路由和部分分方案，不写长题解。

交付标准：

- 重点训练“先拿分，再升级”的动作。

所有模块必须使用同一格式，便于翻阅：

```text
模块编号：
模块名称：
一句话用途：
什么时候用：
不要什么时候用：
复杂度：
数据范围参考：
依赖的标准容器：
可拼接模块：
输入如何整理：
接口：
模板代码：
调用示例：
常见坑：
暴力/部分分替代：
升级方向：
最小测试样例：
```

模板约束：

- C++17。
- 尽量自包含。
- 图默认 1-index。
- 不使用 `#define int long long`。
- 不使用 `freopen`。
- 不使用 `#pragma GCC optimize`。
- 不依赖本地文件。
- 所有距离、答案、计数默认优先考虑 `long long`。

## 5. 模块编号体系

建议编号如下：

- `OPS-xx`：考场策略。
- `CPP-xx`：C++/STL。
- `BRUTE-xx`：暴力、枚举、搜索、部分分。
- `DP-xx`：动态规划。
- `DS-xx`：数据结构。
- `GRAPH-xx`：图论。
- `MATH-xx`：数学。
- `STR-xx`：字符串。
- `ROUTE-xx`：综合路由。

编号收益：

- 页眉和索引可直接引用模块编号。
- 考场可以先看路由表，再按编号翻模块。
- 方便多 Agent 并行写作后合并。

## 7. 主 Agent 合并流程

1. 收集各 Agent 输出。
2. 统一模块编号和标题。
3. 删除重复模板，保留短版/完整版分层。
4. 给每个模块补齐缺失字段。
5. 生成全局索引。
6. 生成核心版和完整版。
7. 抽取 C++ 代码块做编译烟测。
8. 若生成 PDF，渲染抽查页面，检查表格、代码块、页眉页码。

## 8. 质量门槛

每个模块必须通过以下检查：

- 有明确使用场景。
- 有明确“不适用场景”。
- 有复杂度。
- 有数据范围参考。
- 有依赖的标准容器。
- 有可拼接模块说明。
- 有 C++17 模板。
- 有调用示例。
- 有常见坑。
- 有暴力/部分分替代。
- 代码不含禁止项。
- 代码风格能独立抄写，不依赖大型自定义库。

全资料必须通过以下检查：

- 第 -1 卷定义的接口在所有模块中保持一致。
- 第 0 卷可独立指导考场流程。
- 路由表能覆盖主流题型。
- DP 卷不是教材式讲解，而是状态设计字典。
- 暴力卷足够强，能支撑部分分策略。
- 模块索引能在 30 秒内定位到目标模块。
- 核心版不被低频高级算法污染。
- 完整版宁滥勿缺，高级模块放后面。

## 9. 页面与打印设计

建议：

- A4 纵向为主。
- 代码使用等宽字体。
- 表格尽量不跨页。
- 每页页眉包含卷名和模块编号。
- 每个模块起始处突出“什么时候用”和“不要什么时候用”。
- 高频路由表可单独打印并放最前面。
- 完整版厚资料用分页签：作战、C++、暴力、DP、数据结构、图论、数学字符串、综合路由。

不建议：

- 长篇证明。
- 大段故事化解释。
- 只有代码没有调用示例。
- 多套风格不同的同类模板混在一起。
- 把最大流、AC 自动机等低频内容放在最前面。

## 10. 风险与控制

### 风险 1：资料太厚，考场找不到

控制：

- 第 0 卷和路由表必须短。
- 模块编号和页码索引必须完整。
- 产出核心版和完整版两套。

### 风险 2：模板会抄但不会接题

控制：

- 每个模块都必须有“输入如何整理”和“调用示例”。
- 综合路由卷训练题面到模块的映射。

### 风险 3：DP 仍然不知道怎么设状态

控制：

- DP 卷第一页固定为 DP 五问。
- 每类 DP 都给状态形状和典型触发词。
- 强制提供暴力 DFS -> 记忆化 -> 表推的转换路径。

### 风险 4：部分分策略不足

控制：

- 暴力卷作为独立大卷。
- 每个模块都有部分分替代。
- 第 0 卷包含每题版本升级策略。

### 风险 5：模板代码不符合考场限制

控制：

- 自动扫描禁止项。
- 抽取代码块做 C++17 编译烟测。
- 禁止文件 IO 和非标准库依赖。

## 11. 建议讨论点

执行前建议确认以下 6 个问题：

1. 最终资料是否只要 Markdown，还是同时要 PDF 打印版？
2. 需要“核心版 + 完整版”两套，还是只要完整版？
3. 代码模板是否统一全部使用 1-index 图和 `long long` 答案策略？
4. 是否接受高级低频模块放在最后，只保证“有”，不保证第一页高频可见？
5. DP 卷是否要比其他卷更厚，作为重点准备对象？
6. 是否要加入 10-20 个“只训练路由和部分分”的模拟题面摘要？

## 12. 推荐默认答案

如果没有额外偏好，建议采用：

- 输出 Markdown + PDF。
- 同时生成核心版和完整版。
- C++17，图默认 1-index。
- 高频模块放前，高级模块放后。
- DP 和暴力卷加厚。
- 加入综合路由训练。
- 执行阶段启动 8 个 gpt-5.5 xhigh 子 Agent 并行，主 Agent 负责合并和质检。
