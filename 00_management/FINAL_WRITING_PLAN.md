# 最终文档编写计划 v0.1

## 1. 总目标

生成一套面向北京中关村学院夏令营机考的 C++17 纸质开卷资料包。

使用者前提：

- 算法基础薄弱。
- 考试临近，没有时间系统学习原理。
- 需要根据题面场景快速路由、套模块、拼接、提交部分分、再升级。

资料定位：

```text
不是算法教材。
不是孤立模板合集。
而是纸质版算法作战系统。
```

核心动作：

```text
读题 -> 看数据范围 -> 看题面信号 -> 找食谱 -> 整理成标准容器 -> 拼模块 -> 先交部分分 -> 升级
```

## 2. 最终卷结构

### 第 -1 卷：统一接口与标准容器

目的：让所有模块能拼接。

内容：

- 全局 C++17 头部和类型约定。
- 图/数组/区间/查询/状态的统一索引规则。
- 标准 `Graph`：`G.g` + `G.edges` 双视图。
- 标准数组：`vector<ll> a(n + 1)`。
- 标准 `Query`。
- 标准 `Compressor`。
- 标准 `State = {进度, 资源, 记忆}`。
- 统一函数名：`init/build/add/add_edge/setv/range_add/query/prefix/at`。

### 第 0A 卷：题面路由与拼接食谱

目的：解决“用哪个模块、怎么接、怎么验”。

内容：

- 题面信号 -> 模型 -> 主模块 -> 辅助模块。
- 数据范围 -> 复杂度 -> 可用算法。
- 操作类型 -> 数据结构。
- 常见食谱：
  - 静态区间和。
  - 点修改区间和。
  - 区间修改。
  - 无权/有权最短路。
  - Kruskal。
  - DAG DP。
  - 树上路径。
  - 背包。
  - 区间 DP。
  - 状压访问所有点。
- 每个食谱必须有：判断信号、模块组合、拼接顺序、伪代码、部分分版本、升级版本、最小验错。

### 第 0 卷：考场作战手册

目的：考试时保持节奏。

内容：

- 规则摘要。
- 3 小时分配。
- 每题 32 次提交策略。
- 复杂度预算表。
- 先交部分分再升级策略。
- 提交前检查清单。
- 完全不会时的合法兜底输出。

### 第 1 卷：C++17/STL 快查

目的：降低语言层面失误。

内容：

- 输入输出。
- `vector/string/pair/tuple`。
- `sort/lambda/lower_bound/upper_bound`。
- `queue/deque/stack/priority_queue`。
- `set/map/unordered_map`。
- `bitset` 和位运算。
- 坐标压缩。
- 溢出处理。

### 第 2 卷：暴力、记忆化与部分分

目的：每题先活下来，并提供暴力到中档分的升级路径。

内容：

- 全排列。
- 组合枚举。
- 子集枚举。
- 子集的子集枚举。
- DFS 回溯。
- 剪枝。
- 记忆化搜索核心章节：
  - 什么时候用。
  - 状态能不能缓存的判断。
  - 暴力 DFS -> memo 固定改造流程。
  - `vector` memo。
  - `map<tuple,...>` memo。
  - `unordered_map` 编码 memo。
  - 最大值/最小值/方案数/可行性四套返回值模板。
  - 有环状态风险。
  - 与 DP 卷的关系。
- BFS 状态搜索。
- 折半枚举。
- 小数据精确 + 大数据兜底。

### 第 3 卷：DP 模型复用系统

目的：初学者友好地复用已有 DP 模型，匹配不上时从 DFS 造模型。

前置三页：

1. DP 路由表。
2. 状态句式库。
3. DFS -> 记忆化 -> 表推 DP 升级图。

内容：

- 线性 DP。
- 选/不选 DP。
- 0/1 背包。
- 完全背包。
- 分组背包。
- LCS。
- 编辑距离。
- LIS。
- 网格 DP。
- 区间 DP。
- 树形 DP。
- DAG DP。
- 状压 DP。
- 数位 DP。
- 记忆化 DFS 与表推转换。
- DP + 前缀和/树状数组/线段树/单调队列优化。

每个模型必须包含：

- 题面触发词。
- 数据范围信号。
- 状态句式。
- 初始化。
- 转移模板。
- 答案位置。
- 循环顺序。
- 暴力 DFS 版本。
- 记忆化版本。
- 表推版本。
- 常见坑。
- 可拼接模块。

### 第 4 卷：数据结构

目的：解决动态维护、区间、统计类题。

内容：

### 第 5 卷：图论与树

目的：所有图论模块尽量共用 `Graph`。

### 第 6 卷：数学与字符串模板

目的：覆盖常见数论、组合、字符串模板，偏“可直接抄代码”的模块。

内容：

- gcd/lcm。
- 快速幂。
- 模运算。
- 逆元。
- 组合数。
- 筛法。
- 质因数分解。
- 矩阵快速幂。

- AC 自动机/Manacher 低优先级补充。

### 第 6A 卷：竞赛数学知识参考

目的：作为 NOI/ICPC 风格竞赛数学的纸质查阅卷，补足公式、结论、题型路由和常见建模，不只放模板代码。

内容：

- 整除、同余、模运算规则。
- gcd/lcm 与贝祖定理。
- 扩展欧几里得与线性同余方程。
- 模逆元、CRT、扩展 CRT。
- 素数筛、质因数分解、约数个数/约数和。
- 欧拉函数。
- 莫比乌斯/容斥基础作为备用参考。
- 组合数学：排列组合、二项式、Lucas、Catalan、容斥、鸽巢原理。
- 递推与矩阵快速幂。
- 概率与期望基础。
- 博弈论：必胜/必败、SG 函数。
- 基础计算几何公式。
- 浮点误差、整除取整、上下取整。
- 数学题路由表：题面触发词 -> 数学工具 -> 模板。

每节必须包含：

- 题面触发词。
- 使用条件。
- 公式/结论。
- C++17 模板或计算方式。
- 常见坑。
- 暴力/部分分替代。
- 最小验错。

### 第 7 卷：调试、反例与模拟路由

目的：考前训练“看题路由”和“提交前自检”。

内容：

- 常见 WA/RE/TLE 清单。
- 极端样例构造表。
- 10-20 个题面摘要，只训练路由、模块组合和部分分方案。
- 每题不写长题解，重点是“怎么判断、怎么先交、怎么升级”。

## 4. 代码统一约束

- C++17。
- 统一骨架使用 `using namespace std;`，服务 ACM/机考单文件快速编码。
- 不使用 `#pragma GCC optimize`。
- 不使用 `freopen`。
- 不使用文件 IO。
- 不使用 `#define int long long`。
- 图默认 1-index。
- 数组默认 1-index。
- 区间默认闭区间 `[l, r]`。
- 字符串默认 C++ 0-index，必要时显著标注。
- 权值、距离、答案、计数优先使用 `long long`。
- 模块尽量自包含，但遵守统一接口。

## 6. 质检门槛

必须检查：

- 是否所有模块都有触发词和“不适用场景”。
- 是否所有模块声明依赖标准容器。
- 是否所有模块有调用示例。
- 是否所有模块有部分分替代。
- 是否第 2 卷的记忆化章节足够完整。

- 是否代码不含禁止项。
- 是否 C++17 可编译。
